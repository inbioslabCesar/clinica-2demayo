1. Verificar columna:

   FROM INFORMATION_SCHEMA.COLUMNS
   WHERE TABLE_NAME = 'cotizaciones'
     AND COLUMN_NAME = 'responsable_farmacia_id';
   
   Resultado esperado: 1 fila con INT, YES

2. Verificar índices:
   SHOW INDEX FROM cotizaciones 
   WHERE Column_name = 'responsable_farmacia_id';
   
   Resultado esperado: 2 filas

3. Ver estadísticas:
   SELECT 
       COUNT(*) as total_cotizaciones,
       SUM(CASE WHEN responsable_farmacia_id IS NOT NULL THEN 1 ELSE 0 END) as con_responsable,
       SUM(CASE WHEN responsable_farmacia_id IS NULL THEN 1 ELSE 0 END) as sin_responsable
   FROM cotizaciones;

===============================================================================
📞 SUPPORT & TROUBLESHOOTING
===============================================================================

Problema: "Duplicate column error"
Solución: La columna ya existe. Es SEGURO ignorar este error.

Problema: "Can't create index on duplicate keys"
Solución: El índice ya existe. Es SEGURO ignorar este error.

Problema: "Access denied for user"
Solución: Usar usuario con permisos ALTER TABLE (típicamente root)

Problema: "Syntax error near responsable"
Solución: Verificar que MySQL versión >= 5.7

Problema: "Timeout executing query"
Solución: Ejecutar vía MySQL CLI en lugar de phpMyAdmin
